# 논문분석기 결제 시스템 구현 계획

> ⚠️ **작업 지침**: 유료결제 기능은 토큰 한도로 인해 단계적으로 처리. 세션마다 이 파일을 먼저 읽고 현재 단계를 이어서 진행할 것.

---

## 1. 회원 등급 구조

| 등급 | 사용 한도 | 서고 | 결제 |
|------|----------|------|------|
| 비회원 | 1회 무료 (샘플) | ❌ | - |
| 무료회원 | 하루 3회 | ✅ | - |
| 유료회원 (크레딧) | 1 credit = 10MB = 1,000원 | ✅ | 크레딧 충전 |
| 유료회원 (구독) | 월 9,900원 → 10 credit (100MB) | ✅ | 월정액 |

> **크레딧 계산 기준**: PDF 파일 용량 기준 (10MB = 1,000원 = 1 credit)

## 2. 결제 구현에 필요한 전체 작업 목록

### PHASE 1 — 사전 준비 (개발 외 작업, 직접 진행 필요)
- [ ] **사업자등록증** 발급 (개인사업자 or 법인)
- [ ] **통신판매업 신고** (인터넷 쇼핑몰 등록 — PG사 계약 필수 요건)
- [ ] **사업용 은행 계좌** 개설 (PG사 정산 수취용)
- [ ] **PG사 계약** → 추천: **토스페이먼츠** or **포트원(PortOne/아임포트)**
  - 토스페이먼츠: https://developers.tosspayments.com (API 품질 좋음, 스타트업 친화)
  - 포트원: https://portone.io (여러 PG 통합 미들웨어, Next.js 지원 좋음)

### PHASE 2 — 보안 설정 (기술 작업)
- [ ] **HTTPS 강제** (Vercel 기본 제공 ✅)
- [ ] **환경변수 보안 관리** — PG API 키는 반드시 서버 환경변수에만 저장
- [ ] **Webhook 서명 검증** — PG사가 보내는 결제 완료 알림 위변조 방지
- [ ] **RLS(Row Level Security)** — payments 테이블 본인 데이터만 접근
- [ ] **서버사이드 금액 검증** — 클라이언트가 보낸 금액 그대로 믿지 말 것 (서버에서 재계산)

### PHASE 3 — 백엔드 구현 (기술 작업)
- [ ] `app/api/payments/create-order/route.ts` — 결제 주문 생성
- [ ] `app/api/payments/confirm/route.ts` — 결제 최종 승인 (서버사이드)
- [ ] `app/api/payments/webhook/route.ts` — PG사 Webhook 수신 및 크레딧 지급
- [ ] Supabase `payments` 테이블 연동 (이미 생성됨)
- [ ] Supabase `profiles.credits` 업데이트 로직

### PHASE 4 — 프론트엔드 구현 (기술 작업)
- [ ] `/pricing` 페이지 결제 버튼 연동
- [ ] 토스페이먼츠 or 포트원 SDK 설치 및 결제창 호출
- [ ] 결제 성공/실패 리다이렉트 처리
- [ ] 크레딧 잔액 표시 UI

### PHASE 5 — 테스트 및 검수
- [ ] 테스트 결제 (PG사 샌드박스 환경)
- [ ] 환불 처리 플로우 테스트

## 3. 추천 PG사: 토스페이먼츠

### 이유
- 스타트업 친화적, 계약 빠름
- Next.js SDK 공식 지원
- 대시보드 직관적

### 수수료
- 신용카드: 약 2.2~3.3% (계약 규모에 따라 협상 가능)
- 계좌이체: 약 500원/건

### 개발 흐름 (토스페이먼츠 기준)
```
[클라이언트] 결제 버튼 클릭
    → [서버] /api/payments/create-order → orderId, amount 생성
    → [클라이언트] 토스 결제창 호출 (SDK)
    → [사용자] 카드 입력 및 결제
    → [토스 서버] 결제 완료 → 클라이언트로 paymentKey 반환
    → [클라이언트] /api/payments/confirm 호출 (paymentKey + orderId + amount)
    → [서버] 토스 API로 금액 재검증 후 최종 승인
    → [서버] Supabase credits 업데이트
    → [클라이언트] 결제 완료 화면
```
